        ✔ Implements: Queue, Deque, List

        ✔ Order: FIFO (Queue Mode) or LIFO (Stack Mode)

Queue<Integer> queue = new ArrayDeque<>();
queue.add(10);
queue.add(20);
queue.add(30);
System.out.println(queue.poll()); // 10 (FIFO)


Queue<Integer> queue = new PriorityQueue<>();
ANS:    offer() / add(): O(log n)
        poll() / remove(): O(log n)
        peek(): O(1)

        ✔ Use Case: When you need priority-based retrieval, not FIFO.
        ✔ Implements: Queue
        ✔ Underlying Data Structure: Binary Heap
        ✔ Order: Elements are ordered based on priority (Natural Ordering or Custom Comparator).

USING MIN HEAP:
    Queue<Integer> queue = new PriorityQueue<>();
    queue.add(30);
    queue.add(10);
    queue.add(20);
    System.out.println(queue.poll()); // 10 (smallest first)

USING MAX HEAP:
    Queue<Integer> queue = new PriorityQueue<>(Collections.reverseOrder());
    queue.add(30);
    queue.add(10);
    queue.add(20);  
    System.out.println(queue.poll()); // 30 (largest first)



SUMMERY:
    ~ Use LinkedList if you need a doubly linked list with deque functionality.
    ~ Use ArrayDeque if you need a fast, memory-efficient queue.
    ~ Use PriorityQueue if you need elements sorted based on priority.
🚀 Best Practice: Prefer ArrayDeque over LinkedList for a standard queue due to better performance.
